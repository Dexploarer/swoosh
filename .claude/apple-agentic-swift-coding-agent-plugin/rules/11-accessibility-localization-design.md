# 11 — Accessibility, Localization, and Design

## Accessibility

- VoiceOver labels/hints/traits for custom controls.
- Dynamic Type-safe layouts.
- Keyboard navigation where applicable.
- Sufficient contrast and reduced-motion behavior.
- Accessibility identifiers for UI tests.
- App Store accessibility feature notes if relevant.

## Localization

- String catalogs for UI, App Intents, shortcuts phrases, permission strings, error messages, and model-facing user-visible templates.
- Locale-aware dates, numbers, measurements, currency, calendars, and pluralization.
- Do not hardcode prompt text shown to users.

## Design

- Native controls and platform conventions.
- SwiftUI previews for key states.
- Clear AI affordances: what will happen, what data is used, whether output is draft/suggestion, and how to undo.
